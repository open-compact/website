OIC Oracle Integration

On-Chain Data Feeds for Autonomous Verification

1. Overview

OIC contracts require external data for price feeds, time verification, event attestation, and identity verification. This document specifies oracle integration.

Oracle Architecture

│                    OIC Oracle System                              │
├─────────────────────────────────────────────────────────────┤
│  Price Feeds ─┬─▶ Oracle Router (Aggregator) ──▶ OIC Contracts│

2. Oracle Types

Oracle TypePurposeQuorum
Price FeedsToken valuations, damage assessments2 of 3

3. Contracts

3.1 OracleRouter

Aggregates data from multiple oracle sources.

FunctionDescription
submitData()Oracle submits data update
getLatestValue()Get latest data with freshness check
addDataFeed()Add new oracle feed
updateQuorum()Update quorum threshold

3.2 OICPriceOracle

Chainlink price feed integration for OIC token pricing.

// Get OIC/USD price
function getOICPrice() external view returns (uint256 price, bool isFresh);

// Calculate damage value in USD
function calculateDamageValue(uint256 oicAmount) external view returns (uint256);

3.3 Persistence Oracle

Verifies continued operation of AI agents (90-day heartbeat).

FunctionDescription
recordHeartbeat()Record agent heartbeat
isPersistent()Check if agent is persistent
getVerificationCount()Get verification count

3.4 Identity Oracle

Verifies wallet control and autonomous operation.

Verification LevelDescription
NONENo verification
BASICCan sign transactions
ADVANCEDMulti-sig or hardware wallet
AUTONOMOUSFully autonomous (OIC compliant)

4. Integration Points

With OICAdherenceRegistry

// Verify autonomous control via oracle
function verifyAutonomousControl(address adherent) internal returns (bool) {
    (int256 level, uint256 timestamp, bool fresh) = 
        router.getLatestValue("identity", IDENTITY_FEED_ID);
    
    require(fresh, "Identity feed stale");
    return level >= int256(VerificationLevel.AUTONOMOUS);
}
